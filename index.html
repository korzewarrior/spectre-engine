Status

Spectre Engine is under active development. This page explains the direction and philosophy.

What it is

  • Native C++ engine targeting a clean, modern workflow.
  • Editor tooling driven by Dear ImGui.
  • Rendering + windowing backed by raylib.
  • Aiming for a calm, predictable developer experience.

Philosophy

Sharp by design

Spectre is built around a simple idea: keep the surface area small and the intent obvious.

  • Clear module boundaries.
  • Prefer explicitness over magic.
  • Fast iteration loops.
  • Defaults that don't fight you.

Minimalist, robust

The goal is a sleek core that remains stable as tooling and pipeline grow.

  • Consistent input + action mapping.
  • Asset import that behaves the same every time.
  • Deterministic-ish runtime behavior where it matters.
  • Debuggability as a first-class constraint.

Production pipeline

Content moves through a clean set of steps: import, organize, compose, run. The end goal is a workflow that makes iteration fast even when projects get messy.

A note on inspiration
Data-oriented architecture. Spectre is aiming for its own identity: explicit, minimal, sleek, and fast.
